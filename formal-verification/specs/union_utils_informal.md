## Target 1: `Union<'a>` struct

### Source location
`src/util.rs`, lines 122–157.

### Purpose
`Union` is a convenience struct that presents a **read-only logical union** of two
`HashSet<u64>` references.  It is used in the Raft implementation to iterate or query over
two voter sets (the current/incoming configuration and the outgoing configuration) as if
they were one set, without allocating a new combined container.

### Fields
| Field    | Type             | Meaning                                 |
|----------|------------------|-----------------------------------------|
| `first`  | `&HashSet<u64>`  | primary (incoming) set of node IDs      |
| `second` | `&HashSet<u64>`  | secondary (outgoing) set of node IDs    |

### Methods and their specifications

#### `new(first, second) → Union`
**Postcondition**: returns a `Union` wrapping the two references unchanged.

#### `contains(id) → bool`
**Postcondition**: returns `true` **if and only if** `id ∈ first ∨ id ∈ second`.
Equivalent to membership in the mathematical set `first ∪ second`.

#### `is_empty() → bool`
**Postcondition**: returns `true` **if and only if** `first` and `second` are both empty.
Equivalent to `first ∪ second = ∅`.

#### `len() → usize`
**Postcondition**: returns `|first ∪ second|`, i.e. the number of **distinct** elements
across both sets.

**Implementation formula**: `first.len() + second.len() - second.intersection(first).count()`

This is the inclusion-exclusion principle:
```
  |A ∪ B| = |A| + |B| − |A ∩ B|
```
The comment notes "Usually, second is empty" — when `second = ∅` the subtraction term is 0
and `len()` reduces to `first.len()`.

**Invariant**: `len() ≥ 0` (always, since `|A ∩ B| ≤ |A| ≤ |A| + |B|`).

#### `iter() → impl Iterator<Item = u64>`
Yields the **distinct** elements of `first ∪ second`.
*(Not modelled in the Lean file — iterators require a different abstraction.)*

### Key invariants
1. `contains(id) = true ↔ id ∈ (first ∪ second)` (membership correctness)
2. `is_empty() = true ↔ (first ∪ second) = ∅` (emptiness correctness)
3. `len() = |(first ∪ second)|` (cardinality correctness)
4. `len() = 0 ↔ is_empty() = true` (cardinality-emptiness agreement)
5. `contains(id) → len() > 0` (membership implies non-empty)
6. If `second = ∅` then `len() = first.len()` (degenerate case always safe)
7. If `first = second` then `len() = first.len()` (fully overlapping sets)

### Edge cases
- Both sets empty: `contains` returns false, `is_empty` returns true, `len` = 0.
- `first = second` (duplicate set): len equals either set's cardinality.
- One set strictly contains the other: `|A ∪ B| = max(|A|, |B|)`.
- Completely disjoint sets: `len = first.len() + second.len()`.

### Inferred intent
The struct is purely a query interface — it never mutates the underlying sets.  It exists
to allow joint-quorum code to iterate over both voter sets without allocating a third
`HashSet`.  The correctness property that matters most in the Raft context is PROP-3
(`len_correct`): joint-quorum calculations use `len()` as the denominator for majority
arithmetic, so it must equal `|first ∪ second|`.

### Open questions
None — all three methods have clear, unambiguous semantics.

## Target 2: `is_continuous_ents(msg, ents) → bool`

### Source location
`src/util.rs`, lines 78–85.

### Purpose
Checks whether the entries `ents` are **contiguous** with the entries already in `msg`.
Concretely: if both `msg.entries` and `ents` are non-empty, then `ents[0].index` must equal
`msg.entries.last().index + 1`.

### Specification

**Preconditions**: `msg` and `ents` are well-formed (indices increase monotonically; not
enforced by this function but assumed by callers).

**Postconditions**:
- If `msg.entries.is_empty()` → returns `true` (vacuously continuous).
- If `ents.is_empty()` → returns `true` (nothing to append; vacuously continuous).
- Otherwise → returns `msg.entries.last().index + 1 == ents.first().index`.

**Key property**: the function is a **pure predicate** with no side effects.

### Invariants / derived facts
1. `isContinuous [] ents = true` for any `ents`.
2. `isContinuous msg [] = true` for any `msg`.
3. `isContinuous [e1] [e2] ↔ e2.index = e1.index + 1`.
4. Transitivity: if `isContinuous msg ents₁` and `isContinuous (msg ++ ents₁) ents₂`,
   then appending `ents₁ ++ ents₂` to `msg` produces a contiguous sequence.

### Edge cases
- Both empty → true.
- Single-element lists → checks the index gap.
- Large gap (> 1) → false.
- Same index (gap 0) → false (entries would overlap).

### Inferred intent
Used when batching append messages to verify that new entries immediately follow the last
already-included entry, preventing duplicate or gap entries in a batched `MsgApp`.

### Open questions
- Does "continuous" imply the **term** must also be compatible?  No — the function only
  checks index continuity; term compatibility is checked separately.
