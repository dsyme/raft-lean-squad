# Correspondence Tests: `read_only`

This directory documents correspondence validation between the Lean 4 model
`ReadOnly` (in `FVSquad/ReadOnly.lean` and `FVSquad/ReadOnlyCorrespondence.lean`)
and the Rust implementation `ReadOnly` (in `src/read_only.rs`).

## Purpose

Task 8 (Route B) of the Lean Squad workflow verifies that the abstract Lean model and the
concrete Rust implementation agree on observable input/output behaviour for a set of
representative cases. Agreement is checked at two levels:

| Level | Where checked | Tool |
|-------|--------------|------|
| Lean model | `FVSquad/ReadOnlyCorrespondence.lean` | `#guard` (compile-time, 14 cases) |
| Rust implementation | `src/read_only.rs` `test_read_only_correspondence` | `cargo test` (15 cases) |

## Running the tests

```bash
# Lean – compile-time guards (run from repo root)
cd formal-verification/lean
lake build FVSquad.ReadOnlyCorrespondence

# Rust – unit test
cargo test test_read_only_correspondence --features protobuf-codec
```

## Correspondence model

The Lean model `ReadOnly` is a functional abstraction of the Rust `ReadOnly` struct.

| Lean | Rust | Level | Notes |
|------|------|-------|-------|
| `Ctx = Nat` | `ctx: Vec<u8>` | abstraction | Lean k ↔ Rust `vec![k as u8]` |
| `ReadIndexStatus.acks: List Nat` | `ReadIndexStatus.acks: HashSet<u64>` | exact | ordering differs; tests compare sorted |
| `addRequest` | `ReadOnly::add_request` | exact | idempotent guard + append semantics |
| `recvAck` | `ReadOnly::recv_ack` | exact | set-insert into ack set |
| `advance` | `ReadOnly::advance` | exact | FIFO dequeue prefix up to ctx |
| `lastPendingRequestCtx` | `last_pending_request_ctx` | exact | queue tail |
| `pendingReadCount` | `pending_read_count` | exact | queue length |

## Known abstractions

- `ReadIndexStatus.req: Message` (Rust) is elided in the Lean model; only `index` and `acks` are modelled.
- The `logger` parameter of `advance` is omitted; the fatal-panic path for missing ctx is modelled as no-op.
- `ReadOnlyOption` (Safe vs LeaseBased) is not modelled — only affects call sites, not data structure invariants.
- Rust `acks: HashSet` is unordered; Lean `acks: List Nat` is ordered by insertion. Tests compare sorted ack lists.
